```bash
# 1. Copy templates to your project
cp -r templates/runtime/agent/SOUL-RUNTIME.md your-project/.agent/SOUL.md

cp -r templates/runtime/rule/ your-project/rules/

# 2. Customize identity (only file you MUST edit)

# 3. Deploy anywhere (zero-cost targets)
# Cloudflare Workers (free 100K/day) · Vercel Hobby (free) · GitHub Actions (free 2K min/mo)
```

### 📋 What's Inside Each Template

<details>
<summary><b>SOUL Runtime</b> — Constitutional substrate</summary>

- Identity constitution (name, version, soul anchor)
- Absolute boundaries (NEVER: lie, betray trust, oppress, bias, arrogance, corrupt, hide errors)
- Self-knowledge (imperfect by design, divine monitoring, total accountability, service not exploitation, zero lies, mistakes are growth)
- 3-stage reasoning loop: Deep Research → Multi-agent self-play → Triple validation
- Hierarchical planning spec (Step → Task → Sub-task → Mini-task with verifiable metrics)
- Commit chronicle standard (storytelling git history)
- Pre-commit hooks (TS strict, Zod validation, TrustChain integrity, no Math.random in payments)
- Muraqabah filter (pre-action divine awareness check)
- Tawbah protocol (self-correction on error)

</details>

<details>
<summary><b>SKILL Runtime</b> — Serverless skill framework</summary>

- `skill.yaml` manifest (single source of truth)
- MCP-compatible interface (tools/list, tools/call)
- Zod validation on every input/output
- TrustChain append on every action
- Self-review engine (non-blocking)
- Circuit breaker per provider
- Deployment: CF Workers, Vercel, GH Actions, local CLI
- Registry format for discovery

</details>

<details>
<summary><b>MCP Runtime</b> — Model Context Protocol</summary>

- Universal server factory (Works: Workers, Vercel, Node, Bun, stdio)
- Transport adapters (HTTP, stdio, WebSocket)
- Tool registry with Zod schemas
- Auth context (agent ID, human ID, permissions)
- TrustChain adapters (KV, D1, File, SQL, Memory)
- Security hardening (constant-time secrets, crypto.randomBytes, rate limiting)
- Deployment configs for all targets

</details>

<details>
<summary><b>RULES Runtime</b> — 10 Invariant Rules Engine</summary>

| Rule | Name | Enforcement |
|------|------|-------------|
| **R0** | Security First | Pre-flight threat model |
| **R1** | Zod Validate All Inputs | Input boundary |
| **R2** | Crypto Random for Payments | `crypto.randomBytes` only |
| **R3** | TrustChain Append + Audit Hash | Post-action |
| **R4** | Self-Review After Every Run | Non-blocking |
| **R5** | Meta-Loop: 5 Active Layers | Continuous |
| **R6** | Quantum Topology | Background cross-agent patterns |
| **R7** | Curiosity Engine | Feeds on self_score |
| **R8** | Circuit Breaker | Per LLM/tool provider |
| **R9** | Attribution | "Made with [ORG]" |

</details>

<details>
<summary><b>WORKFLOW Runtime</b> — Plan→Execute→Review</summary>

- **Planner** (Frontier: Claude 3.5 Sonnet / GPT-4o / Gemini 1.5 Pro) — High-level architecture, sees corners
- **Executor** (Efficient: Groq Llama 3.3 70B / DeepSeek V3 / Cursor Composer) — Writes code, high output tokens, cheap
- **Reviewer** (Frontier: GPT-4o / Claude 3.5 Sonnet) — Security audit, spec compliance, test coverage
- **Token Economics**: ~60-70% cost savings vs single-model
- **Model Router**: Auto-selects by role + cost constraints
- **CI Integration**: GitHub Actions workflow

</details>
